# POS オフライン会計

Android Store App の `/store/pos` は、オンラインで一度読み込んだ店舗について最大24時間、現金の応急会計を行えます。

## 対応範囲

- 現金の店内・持ち帰り注文
- 端末内への注文保存
- Bluetooth / USB / Star プリンターへのレシート・厨房票印刷
- 通信復旧後の自動同期
- クライアント注文IDによる重複防止

オフライン中は会員、ポイント、クーポン、割引、テーブル会計、オンライン決済を利用できません。未同期注文がある間は日次レジ締めを実行できません。

## 保存と同期

POS のメニュー、価格、店舗設定、レジ状態は IndexedDB に店舗別スナップショットとして保存されます。オフラインで開始した注文には UUID と `O` で始まる取扱番号を付け、印刷物には `オフライン未同期` と表示します。通信中断時の二重登録も防ぐため、通常の POS 会計も最初の送信から同じ UUID を使います。

同期APIは `offlineClientOrderId` を `store_customer_orders.offline_client_order_id` に保存します。同じIDが再送された場合は既存注文を返すため、通信切断やアプリ再起動による二重登録を防ぎます。

同期時に現在のメニュー金額と会計時金額が異なる場合は自動登録せず、POS に管理者確認エラーを表示します。

## アプリシェル

Service Worker は、オンラインで利用した `/store` 画面と対応する Next.js 静的ファイルを保存します。Android App を完全にオフラインで再起動するには、対象バージョンへの更新後に一度オンラインで `/store/pos` を開いてください。
