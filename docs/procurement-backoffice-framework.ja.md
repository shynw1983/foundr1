# 店舗仕入れ管理システム構想

## 目的

複数店舗・複数ブランドの日常仕入れを一元管理するためのバックオフィスです。店舗スタッフが必要商品を申請し、仕入れ担当が購入状況、欠品、価格異常、配送状況を記録し、店舗側が受け取り確認できる流れを作ります。

現時点では招商・加盟向けのフロントサイトは優先しません。ただし将来的にブランドサイト、加盟募集ページ、店舗展開ページを追加しやすいよう、Next.js を前提に構築します。

## 第一版で扱う対象

- 店舗管理
- ブランド管理
- 担当者別の閲覧・操作権限
- 商品マスタ
- 仕入れ先管理
- 仕入れ依頼作成
- 仕入れ依頼処理
- 欠品・異常報告
- 配送・受け取り確認
- 主要商品の価格記録
- 簡易ダッシュボード

## 権限とデータ分離

従業員の権限は役割だけでなく、所属店舗、担当ブランド、担当範囲を組み合わせて設定します。

主な考え方：

- 本部仕入れ担当は全店舗・全ブランドの仕入れ依頼を確認できる。
- 店舗仕入れ担当は自分に割り当てられた店舗の依頼だけを確認・処理できる。
- ブランド別担当は担当ブランドに関係する依頼だけを確認できる。
- 店舗スタッフは所属店舗の依頼作成、異常確認、受け取り確認を行う。
- 管理者は店舗、ブランド、担当者、権限スコープを管理する。

仕入れ依頼を作成するときは、納品先店舗と適用ブランドを必須にし、その情報をもとに閲覧可能な担当者を自動判定します。これにより、店舗担当と本部担当の情報を分離しながら、必要な依頼だけをそれぞれの画面に表示できます。

権限スコープの例：

- 全社スコープ：全店舗、全ブランド、全仕入れ先を管理。
- 店舗スコープ：指定店舗の依頼、欠品、配送、受け取り確認のみ管理。
- ブランドスコープ：指定ブランドの商品、依頼、価格傾向を管理。
- 仕入れ先スコープ：指定仕入れ先に関係する依頼だけを処理。

## 商品マスタ

対象は食材、包材、消耗品、清掃備品、設備消耗品です。

主な項目：

- 商品名
- カテゴリ
- 適用ブランド
- メイン仕入れ先
- 参考価格
- 標準依頼数量
- 単位
- 規格メモ
- 重要商品フラグ
- 価格変動注意フラグ
- 冷蔵・冷凍などの管理条件
- 商品画像、後続拡張

同じ食材や備品が複数ブランドで使われる場合、商品を重複登録しません。商品本体は 1 件だけ作り、ブランド別利用設定でどのブランドがどう使うかを管理します。

ブランド別利用設定の項目：

- 商品
- ブランド
- 用途
- 標準依頼数量
- 規格メモ
- 優先度
- 表示順
- ブランド別の依頼可否

将来のデータ構造：

- `products`：商品本体
- `product_brand_usages`：商品とブランドの利用関係、適用用途、標準数量

## 商品別仕入れ先

同じ商品を複数の仕入れ先から購入できるため、商品本体と仕入れ先別設定は分けて管理します。

商品マスタは「何を買うか」を管理し、商品別仕入れ先は「どこで、いくらで、どの優先順位で買うか」を管理します。

仕入れ先別に持つ項目：

- 対象商品
- 仕入れ先
- 役割：メイン仕入れ先、予備仕入れ先、臨時仕入れ先
- 参考価格
- 最小購入数量
- リードタイム
- 対応店舗
- 適用ブランド
- 欠品時の代替可否
- 備考

仕入れ処理時はメイン仕入れ先を初期表示し、欠品、価格異常、納期遅延があれば予備仕入れ先または臨時仕入れ先へ切り替えます。実際に購入した仕入れ先、数量、価格、異常理由は仕入れ依頼明細に記録します。

将来のデータ構造：

- `products`：商品本体
- `suppliers`：仕入れ先本体
- `product_supplier_options`：商品と仕入れ先の対応、優先順位、参考価格
- `purchase_order_items`：依頼された商品と数量
- `purchase_actuals`：実際の仕入れ先、購入数量、実績価格、異常内容

## 仕入れ先と拠点

仕入れ先にはネットショップ、単独の実店舗、複数店舗を持つチェーン店があります。そのため、仕入れ先本体と購入拠点を分けて管理します。

仕入れ先本体は会社名、仕入れ先ブランド名、EC サイト名などを管理します。購入拠点は具体的なネットショップ URL、実店舗、チェーン店の分店を管理します。

購入拠点の項目：

- 仕入れ先
- 拠点名
- 種別：ネットショップ、実店舗、チェーン店
- エリアまたは住所
- 営業時間
- 購入方法：配送、店頭購入、店頭受取、電話注文
- 対応店舗
- 配送可否
- 緊急購入可否
- 備考

実際の購入記録では、仕入れ先だけでなく購入拠点まで残します。これにより、同じチェーン店でもどの分店で買ったか、どの店舗から近いか、どの拠点が欠品しやすいかを後から確認できます。

将来のデータ構造：

- `suppliers`：仕入れ先本体
- `supplier_locations`：ネットショップ、実店舗、チェーン分店などの購入拠点
- `product_supplier_options`：商品と仕入れ先、必要に応じて推奨拠点を紐づける
- `purchase_actuals`：実際に購入した仕入れ先と購入拠点を記録

## 仕入れ依頼の流れ

1. 店舗スタッフがカテゴリやテンプレートから商品を選ぶ。
2. 数量、締切時間、優先度、納品先店舗、適用ブランドを入力する。
3. 仕入れ依頼が「仕入れ待ち」になる。
4. 仕入れ担当が購入を進める。
5. 実際の購入数量、欠品、代替商品、価格異常を記録する。
6. 店舗へ購入完了または異常を共有する。
7. 配送後、店舗が受け取りを確認する。
8. 異常がある場合は対応完了まで追跡する。
